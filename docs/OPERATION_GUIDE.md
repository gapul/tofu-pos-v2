# Tofu POS 運用ガイド

学園祭の模擬店オペレータ向け。3 役割（レジ／キッチン／呼び出し）で 1 つの POS システムを構成します。

---

## 共通: 起動〜初期設定（全端末で 1 回）

各端末で **同じ手順**を踏みます。

1. **アプリ起動**
2. **店舗 ID を入力**
   - 例: `yakisoba_2026`（半角英数字、全端末で**完全に同じ文字列**にする）
   - 全角・スペース・大文字小文字に注意。少しでも違うと別店舗扱いで連携できません。
3. **役割を選ぶ**: `レジ` / `キッチン` / `呼び出し` のいずれか
4. **設定 → 機能フラグ**で必要な機能（キッチン連携・呼び出し連携・客層など）が ON になっているか確認
5. 右上に **接続中の端末バッジ**（レジ・料理・呼出のアイコン）が出ます。他端末が起動したら点灯することを確認

> 💡 店舗 ID を変えてログインしなおすと、**前店舗のローカルデータは自動削除**されます（商品マスタ・整理券プールなど）。

## レジ役の手順

### 開店前の準備

1. **レジ締め画面で釣銭準備金を登録**
   - 設定 → レジ締め → 「釣銭準備金を登録」
   - 金種ごとに枚数を入力 → 登録
2. **商品マスタを登録**
   - 設定 → 商品マスタ → 各商品の名前・価格・在庫を入力
   - 登録した商品は自動的にキッチンへ同期されます
3. **右上の接続バッジ**で「料理」「呼出」の端末が点灯しているか確認

### お客さま対応のルーチン

```
お客が来店
   ↓
「次のお客様」ボタンを押す
   ↓
商品を選んでカートに入れる
   ↓
「会計へ進む」
   ↓
お客が出した金額を「預り金（金種別）」で入力
   ↓
お釣りのサジェスト（最少枚数）が右側に出るので、その通りに渡す
   ↓
「会計確定 → 整理券N」を押す
   ↓
（任意）客層を入力（スキップ可能。ただしレジは焦るとミスが増えるので、
       一度落ち着く意味でも入力推奨）
   ↓
3 秒後に商品選択画面に自動で戻る → 次のお客さまへ
```

### 注意

- **預り金入力中の慌てた打ち間違いに注意**。釣銭サジェストが「最少枚数」で出ますが、手元の金種が不足していたら無視して別の組合せで返してください。
- 顧客属性入力は**スキップ可能**ですが、サークルで集計したいデータなので、忙しくない時は入力をお願いします。

## キッチン役の手順

### 開店前の準備

1. **右上の接続バッジ**で「レジ」「呼出」の端末が点灯しているか確認
2. レジから「商品マスタ」が同期済か確認（自動）

### 通常運用

```
レジから注文通知が届く
   ↓
未調理タブに「[ 伝 票 ] 整理券番号 + 注文内容」が表示される
   ↓
料理を作る
   ↓
完成したら該当カードの「提供完了」を押す
   ↓
カードが「提供済」タブに移動
   ↓
呼び出し端末に「呼び出し前 → 呼び出し」の通知が自動で送られる
```

### 注意

- **提供完了ボタンを押し忘れない**。呼び出し端末は提供完了を受けて初めて顧客への呼出を行います。
- 取り消しが必要なら、設定 → 注文履歴 から該当注文を取り消し（レジ側操作）。

---

## 呼び出し役の手順

### 開店前の準備

1. **右上の接続バッジ**で「レジ」「料理」の端末が点灯しているか確認

### 通常運用

```
レジで会計確定 → 「呼び出し前」タブに整理券番号が追加される
   ↓
（心構えをして待つ。料理完成までは何もしない）
   ↓
キッチンが「提供完了」を押すと、画面いっぱいに整理券番号が表示される
   ↓
番号を声に出して呼ぶ
   ↓
右下の「呼び出し済み」ボタンを押す
   ↓
「呼び出し済」タブに番号が移動
   ↓
お客が受け取りに来たら、「呼び出し済」タブの該当番号をタップ
   ↓
「受取完了」になる → 整理券番号は 3 分後に再利用可能になる
```

### 注意

- **ポップアップを誤って閉じてしまった場合**: 下にスワイプ／左下「閉じる」で閉じても、呼び出し済みにはなりません。「呼び出し前」タブの番号をタップで再表示できます。
- **3 分のクールタイム**: 受取完了から 3 分以内は同じ番号が再使用されません。新しいお客に整理券が重複しないようにする仕組みです。

## 通信トラブル時

### Wi-Fi が不安定

- 設定 → 通信モード → 確認（オンラインが基本）
- 「サーバーに再接続」ボタンで強制再同期
- ヘッダーの 🔄 リロードボタンでサーバから最新情報を引き直し

### 一部端末でデータが届かない

- 各端末で **同じ shop_id** か再確認
- 各端末で **同じ機能フラグ** が ON か再確認（キッチン連携・呼び出し連携）
- 受信側端末でヘッダーの 🔄 を押して手動再同期
- 上から下に**プル**で画面を引っ張ってもリフレッシュ

### サーバ完全ダウン時

- BLE フォールバック（実験中）: オンライン不可でも BLE で会計→キッチン→呼出は配信されます（同じ shop_id で物理的に近距離にあること）
- 商品マスタ・釣銭設定の**初回同期は必ずオンライン**で行う必要があります

---

## 役割切替

設定 → 役割を変更
- 同じ shop_id 内で役割を変更します（ローカルデータは保持）
- 別の店舗 ID にしたい場合は「ログアウト」→ 新 shop_id 入力（旧店舗のローカルデータは自動削除）

---

## ログアウト

設定 → ログアウト
- shop_id をクリアし `/setup/shop` に戻る
- ローカル DB と SharedPreferences の機能フラグ等は保持
- 同じ shop_id で再ログインすればデータが復元
